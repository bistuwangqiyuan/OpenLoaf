
## 现状问题

1. **"项目/对话" 双 Tab 割裂信息** — 对话应属于项目，而非独立顶层分类
2. **"AI 助手"地位过低** — 藏在菜单项中，但它本质是管理层入口
3. **没有群组位置** — 跨项目协作无处落脚
4. **临时对话归属不清** — 临时对话在"对话" Tab 里，但它是管理层行为
5. **画布与对话混合** — 在"对话" Tab 中混合显示，缺乏清晰归属

## 新 Sidebar 结构

去掉 "项目/对话" 双 Tab，改为**单列统一视图**，分为四个区域：

### 区域 1：顶部入口

| 项目 | 说明 |
|------|------|
| ✨ AI小助理 | 管理层入口。点击 → 新建管理层对话。⌘Y 快捷键 |
| 🖌️ 画布 | 创意画布入口。点击 → 新建画布 |
| 📊 工作台 | 全局工作台/仪表盘。⌘T 快捷键 |
| 📅📧⏰ | 全局工具横排图标（日历/邮件/任务），带 Badge |

### 区域 2：项目列表（弹性区域，可滚动）

- **[+ ▾] 按钮**：点击弹出类型选择菜单（AI 团队 / 知识库 / 代码仓库 / 数据分析 / 内容创作 / 自定义项目）
- **项目图标**：按类型自动分配（🤖📚💻📊🎨⚙️），用户可自定义
- 支持拖拽排序
- 支持右键菜单（重命名、删除、设置等）

### 区域 3：群组列表（吸底固定）

- **[+] 按钮**：创建新群组，选择关联项目
- 群组固定在底部，不被项目列表滚动影响

### 统一交互模式

所有可展开项（AI小助理、画布、项目、群组）hover 后**右侧弹出浮层**，使用同一套组件，仅数据源不同。

类似飞书/Notion 的侧边预览效果。

### 浮层数据源

| 触发项 | 浮层内容 | 列表项格式 |
|--------|---------|-----------|
| AI小助理 | 所有对话（跨项目） | 对话标题 + 所属项目名 |
| 画布 | 所有画布（跨项目） | 画布标题 + 所属项目名 |
| 某个项目 | 该项目内的对话 | 对话标题 + 时间 |
| 某个群组 | 该群组内的对话 | 对话标题 + 时间 |

### 浮层 UI 规格

- **左侧**：对话/画布标题，超长截断
- **右侧**：所属项目名（灰色辅助文字），不属于项目的显示 `—`
- **懒加载**：首次加载 20 条，滚动到底部继续加载
- **新建按钮**：浮层右上角，快速新建对话/画布

### 点击行为

| 操作 | 行为 |
|------|------|
| 点击菜单项本身（如 ✨ AI小助理） | 新建一个对话 |
| 点击浮层中某条记录 | 直接打开该对话/画布 |
| 点击项目名 | 进入项目视图 |

## 对话归属变化

| 场景 | 现在 | 改后 |
|------|------|------|
| 临时问个问题 | "对话" Tab 里的临时对话 | AI小助理的对话记录（管理层） |
| 项目内讨论 | "对话" Tab（混在一起） | 项目内部的对话列表 |
| 跨项目协调 | 不存在 | 群组对话 |
| 画布 | "对话" Tab（混合显示） | 跟随所属项目，或 AI小助理 |

## 设计规范遵循

实现时须遵循 `web-style-system` skill 中定义的视觉语言：

- 胶囊圆角 `rounded-full`
- 半透明背景 + 细边框，禁止 `box-shadow`
- 语义扁平色（按项目类型着色）
- 动效 0.16~0.24s
- Light + Dark 双套表达
- 浮层使用 `bg-popover` + `border` + 背景模糊
