# Black Industrial Catalog Style DNA

## Positioning

一句话定义：

`黑场工业目录风 = 硬件展品化 + 黑色 editorial 摄影 + 超细工业字体 + 说明书式信息组织`

风格公式：

`黑场 + 单一主对象 + 硬栅格 + 极轻无衬线 + 灰阶层级 + 目录式 CTA + 克制动效`

它不是普通电商页面，也不是互联网科技 landing page。它更像：

- 一本高密度器材目录
- 一页黑色展览海报
- 一个把硬件当成文化对象处理的产品页

## Color System

主色全部围绕黑灰白展开。

推荐色板：

- `#000000` 页面主底
- `#0f0f10` 深黑结构层
- `#f5f5f5` 主文字 / 高亮文字
- `#a1a1a1` 次级说明
- `#e5e5e5` 浅灰卡片底
- `#cbcbcb` 浅灰辅底
- `rgba(255,255,255,0.10)` 细线边界
- `#d92626` 录制 / 警示红
- `#0071bb` 冷蓝辅助

使用原则：

- `90%` 以上面积保持黑灰白。
- 强调色只做语义点缀，不做气氛渲染。
- 不做大面积渐变情绪背景。
- 不靠彩色建立主层级。

## Typography

主基调是超轻、克制、工业化的 grotesk。

字体方向：

- Univers Next
- Suisse Int'l
- Neue Haas Grotesk
- Helvetica Neue
- Nimbus Sans

不要：

- 粗黑标题堆满屏幕
- 默认 SaaS 字体味道太重
- 标题很潮、正文很普通，像两个系统拼起来

排版原则：

- 标题用 `light / thin`
- 正文继续沿用同家族
- 数字和规格可以更窄更硬
- 层级主要靠 `字号 + 间距 + 位置 + 明度`

## Layout

### Structural Rules

- 使用强栅格，桌面优先 `12 列`
- 容器最大宽度优先 `1440px`
- 边距优先 `48px - 72px`
- gutter 优先 `16px`
- 区块垂直节奏优先 `96 / 144 / 192px`

### Composition Tendency

- 顶部更像目录，不像营销 navbar
- Hero 只允许一个主角
- 标题轻、位置准、留白大
- 说明文通常不长，但有明显技术口吻
- 规格区像手册
- 商店区反而用浅灰卡片，形成黑白节奏反差

## Imagery

图像必须“展品化”，不要普通电商化。

优先使用：

- 黑底器材特写
- 近正视或局部结构特写
- 高反差、高控制的边缘光
- 少量黑色时尚 editorial 人像，但仍服务于对象气质

避免：

- 生活方式场景图铺满页面
- 明亮温暖的消费级商业摄影
- 过度 CG 光效
- 悬浮在彩色背景上的产品渲染

## Component Grammar

### Directory Navigation

- 导航像目录索引，不像营销菜单
- 小字号、轻字重、分组明确
- 允许加入几何图形或极简对象图标

### Hero

- 一个对象 + 一个轻标签 + 一个超大标题
- 标题不要太重
- CTA 必须弱，不可喧宾夺主

### CTA

- 像标签、状态块、目录动作
- 可以是黑底胶囊，也可以是灰阶分段块
- 文案极短：`buy now` / `visit store` / `view specs`

### Product Card

- 浅灰大底
- 大圆角但不软萌
- 图片占比高
- 文本像产品目录标签
- 禁止重阴影和悬浮感

### Specs

- 像说明书，不像营销文案段落
- 分类名亮一点
- 参数值暗一点
- 靠留白做分组

## Copy Tone

文案更像：

- 技术宣言
- 设备说明
- 型号标签
- 冷静短句

推荐：

- 短句
- 少形容词
- 动作词和对象词前置
- 把功能写成“设备能力”

避免：

- 重新定义未来
- 革命性体验
- 沉浸式生态
- 互联网热情营销腔

## Motion

推荐规则：

- 时长优先 `180ms - 220ms`
- 只用 `opacity / transform / color`
- hover 只做亮度、对比、轻微上抬
- 大图可做极轻淡入

避免：

- spring 回弹
- 大视差
- blur 发光
- 漂浮感强的连续表演

## Do / Don't

### Do

- 让对象成为视觉主角
- 让页面像硬件目录和展览画册的混合体
- 用黑灰白建立真正层级
- 让按钮像目录动作
- 让规格区像手册

### Don't

- 不要变成泛科技 landing page
- 不要变成普通电商详情页
- 不要过度装饰 UI
- 不要靠颜色替代版式
- 不要把整个页面做得又软又萌
