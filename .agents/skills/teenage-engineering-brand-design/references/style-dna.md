
## Positioning

一句话定义：
`工业说明书式极简 + 编辑海报感 + 北欧功能主义 + 少量玩具化幽默`

风格公式：
`硬网格 + 大留白 + 冷灰黑白底 + 极少量高纯度点色 + 细无衬线 + 产品展品化 + 技术铭牌式文案 + 克制交互`

默认不要把它理解成“科技炫酷风”或“未来主义炫技风”。它更像高设计浓度的硬件品牌系统，而不是互联网产品落地页模板。

## Palette

以中性色为主，强调色极少量出现。

推荐主色板：

- `#0f0e12`：主黑，正文、结构、深色块
- `#f5f5f5`：冷白，产品浅底、模块面
- `#e5e5e5`：浅灰，大面积页面背景
- `#cccccc`：边界灰，弱分隔
- `#a1a7af`：辅助灰，次级信息
- `#f05a24`：橙色强调，首选品牌点火色
- `#0071bb`：蓝色辅助强调，仅局部使用
- `#006837`：绿色辅助强调，仅功能性或系列化场景
- `#fab413`：黄色提醒，仅少量点缀
- `#b81d13`：红色警示，仅语义化使用

使用原则：

- 80% 以上面积保持中性色。
- 强调色不要平均分布，只点在一两个关键部位。
- 不做大面积饱和背景，不做梦幻渐变氛围。

## Typography

主基调是 `thin grotesk`，像工业目录、设备铭牌、现代主义标识系统。

推荐做法：

- 标题：使用轻字重或细字重无衬线
- 正文：继续使用同一字体家族，避免“标题很潮、正文很普通”
- 数字和型号：允许更硬、更技术化的字形
- 特殊系列页：少量混入实验性显示字体，但不能喧宾夺主

可替代字体方向：

- Univers / Neue Haas Grotesk / Haas Unica / Suisse Int'l 风格
- 若无法获取相近商用字体，优先选中性、节制的 grotesk 族

避免：

- 直接复用对方专有字体
- 过度时髦或过度互联网化的默认字体方案
- 粗重标题到处堆满，导致页面失去空气感

### Structural Rules

- 使用强栅格，优先 12 列或同等强度的自定义网格
- 使用大边距和稳定的列宽关系
- 让版式先成立，再决定色块和图
- 导航、目录、注释、说明信息全部服从同一网格

### Spatial Feel

- 留白大，但不是空
- 信息密度不低，只是靠秩序让页面显得冷静
- 常见结构是：
  `品牌头部 / 单一主视觉对象 / 技术标签 / 产品卡片目录 / 规格或叙事段`

### Composition Tendency

- 大对象居中或偏置展示
- 卡片表面常是浅灰大底
- 局部用极细线条图标或轮廓字做结构点缀
- 页面允许显得“有点像展册”或“有点像器材目录”

## Imagery

图像不是普通电商拍法，而是“展品化”和“器材化”。

优先使用：

- 产品正面或近正面陈列
- 中性背景下的高分辨率 packshot
- 强叙事 editorial hero，但构图依然克制
- 对结构、按钮、材料、边缘细节的强调

避免：

- 常规 lifestyle 电商图堆满页面
- 过度情绪化的光效
- 大量三维悬浮特效

### Navigation

- 顶部导航像目录，而不是营销站菜单
- 细字重、小字级、线框图标、低压迫感
- 分类命名短、直接、功能性强

### Hero

- 通常只有一个主对象
- 主对象可配超大轮廓字、型号、简短标签
- 文案少，优先让对象自己说话

### Product Cards

- 大面积浅灰底
- 边界干净，圆角存在但不软萌
- 图片占比高，标题和价格像目录标签
- 状态标签可用小黑胶囊或简洁角标

### CTA

- CTA 克制，不要“大促销按钮”气质
- 常见形态是黑色块、细字、下划线链接、简短动作词
- 优先让 CTA 像“目录动作”而不是“情绪召唤”

### Specs and Labels

- 规格说明像技术清单
- 标签短、直给、编号感强
- 可以混用大小写、数字、型号、符号，形成设备语言

## Motion

动效应当存在，但存在感低。

推荐规则：

- 时长优先 `150ms - 250ms`
- 只用 `opacity / transform / color`
- 滚动动画少而准
- hover 只表达状态变化，不做戏剧化位移

避免：

- 夸张 spring
- 模糊扩散发光
- 大范围视差
- 首屏多元素连续表演

## Copy Tone

文案气质要像：

- 产品编号
- 技术说明
- 轻度调侃
- 冷静自信

推荐：

- 短句
- 少形容词
- 型号、规格、关键词前置
- 把“功能”写得像“设备能力”

避免：

- 大段品牌宣言占满首屏
- 互联网营销腔
- 过度热情或过度煽情

## Implementation Hints

如果要落到 CSS / Design Token，可从下面起步：

实现时优先：

- 先搭网格，再上组件
- 先确定 hero 对象，再决定标签密度
- 先处理黑白灰关系，再打入橙/蓝/绿点色

### Do

- 让页面像精心设计的硬件目录
- 用少量颜色建立记忆点
- 用字重、大小、位置和留白建立层级
- 让产品对象成为视觉主角
- 保持冷静、克制、准确

### Don't

- 不要直接复刻品牌 logo、字体文件、图标资产
- 不要把页面做成泛科技 SaaS 模板
- 不要靠玻璃拟态、重阴影、紫蓝渐变制造“高级感”
- 不要把所有按钮都做成同一强调色大块
- 不要把页面做得太软、太萌、太圆、太 UI kit
